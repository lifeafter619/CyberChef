        X 功能如何工作？

        CyberChef 提供上下文帮助功能。将鼠标悬停在想了解的功能上并按下键盘上的 F1 即可获取说明。现在试试把鼠标悬停在这段文字上并按下 F1！


        CyberChef 可以做哪些事情？

        CyberChef 拥有 数百 个操作，能轻松完成简单或复杂的任务。以下是一些示例：

        • 解码 Base64 编码字符串
        • 将日期时间转换到其他时区
        • 解析 Teredo IPv6 地址
        • 将十六进制转储转换后解压
        • 解密并反汇编 shellcode
        • 将多个时间戳显示为完整日期
        • 对不同类型的数据执行不同操作
        • 将输入的一部分用作操作参数

        可以直接从文件加载输入吗？

        可以！将文件拖到输入框并放下即可。

        CyberChef 可处理约 2GB 的文件（取决于你的浏览器），但部分操作在这么大的数据上可能会运行很久。

        如果输出超过某个阈值（默认 1MiB），它会以可下载文件的形式提供。如需查看，可在输出中查看文件切片。


        如何对多个输入同时运行操作 X？

        例如你有 10 个时间戳需要解析，或 16 个使用同一密钥的编码字符串。

        “Fork” 操作（位于 “Flow control” 分类）会按行拆分输入，并对每行分别执行后续操作。每个输出会显示在独立的行中。分隔符可以更改，如果输入使用逗号分隔，可以将分隔符改为逗号。

        示例请 点击这里。


        “Magic” 操作如何工作？

        “Magic” 操作使用多种方法来检测编码数据以及可能用于解析它的操作。有关这些方法的技术说明可在 这里 查看。

        是什么
        
                                
        一个简单直观的 Web 应用，用于分析和解码数据，无需处理复杂工具或编程语言。CyberChef 鼓励技术人员和非技术人员探索数据格式、加密和压缩。

        

                                
        为什么
        
                                
        在现代世界，数字数据形态各异。CyberChef 帮助你在一个易用的平台上理解这些数据。

        


                                
        如何
        
                                
        界面以简洁为核心。复杂技术如今变得像拖放一样简单。简单函数可以组合成“配方”，从而进行复杂分析，并可与其他用户共享并复用他们的输入。
        
                                
        对于擅长编写代码的人，CyberChef 是快速高效的原型工具，验证可行后可脚本化落地。

        


                                
        适用人群
        
                                
        CyberChef 预计对网络安全与杀毒公司非常有用，也适用于学术界以及任何从事数字数据分析的个人或企业，例如软件开发者、分析师、数学家或解谜爱好者。

        


                                
        目标
        
                                
        我们希望通过在 GitHub 发布 CyberChef，让社区贡献能够融入到该工具的未来版本中。

        


                                
        
                                
        CyberChef 拥有 数百 个实用操作，适用于任何与互联网相关的工作场景，无论你只是想将时间戳转换格式、解压 gzip 数据、生成 SHA3 哈希，还是解析 X.509 证书以找出签发者。
        
                                
        它就是网络瑞士军刀。

            下载 CyberChef
            
                    
            
                    
            
                        
            
                            CyberChef 完全在你的浏览器中运行，没有服务端组件。这意味着无论你使用官方在线版本还是下载的独立版本（前提是未被修改），你的输入数据和配方配置都不会被发送到任何地方。
                        
            
                        
            
                            仅有三个操作会调用外部服务：用于从 wikimedia.org 下载地图瓦片的 “Show on map” 操作、通过 Google 或 Cloudflare 解析 DNS 请求的 “DNS over HTTPS” 操作，以及调用你配置的 URL 的 “HTTP request” 操作。你可以使用浏览器开发者工具（F12）的 Network 选项卡确认具体网络请求。
                        
            
                        
            
                            如果你希望下载 CyberChef 的独立副本在隔离网络或无网络环境中运行，可以在下方获取包含完整 Web 应用的 ZIP 文件。它可本地运行或托管在 Web 服务器上，无需额外配置。
                        
            
                        
            
                            请注意，独立版本不会自动更新，只有在你手动重新下载新版本后，才会获得漏洞修复或新功能。
