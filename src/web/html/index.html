        X 功能如何工作？

        CyberChef 提供上下文帮助功能。只需将鼠标悬停在你想了解的功能上，并按下键盘上的 F1 即可获取说明。现在就将鼠标悬停在这段文字上并按下 F1 试试看！


        我可以用 CyberChef 做什么？

        CyberChef 内置了 数百个 操作，让你轻松完成简单或复杂的任务。以下是一些示例：

        • 解码 Base64 编码字符串
        • 将日期时间转换到不同的时区
        • 解析 Teredo IPv6 地址
        • 从十六进制转储还原数据并解压
        • 解密并反汇编 shellcode
        • 将多个时间戳显示为完整日期
        • 对不同类型的数据执行不同操作
        • 使用输入的部分内容作为操作参数

        我可以直接从文件加载输入吗？

        可以！只需将文件拖到输入框并释放即可。

        CyberChef 可处理约 2GB 的文件（取决于浏览器），不过某些操作在如此大数据量上可能需要很长时间。

        如果输出超过某个阈值（默认 1MiB），它会以可下载文件的形式呈现。如需检查内容，可在输出中查看文件切片。


        如何在多个输入上同时运行某个操作？

        比如你有 10 个时间戳需要解析，或 16 个使用相同密钥的编码字符串。

        “Fork” 操作（位于“Flow control”类别）会按行拆分输入，并对每行分别执行后续操作。每个输出随后会在单独的行中显示。分隔符可调整，例如如果输入以逗号分隔，可将分隔符改为逗号。

        查看示例请 点击这里。


        “Magic” 操作如何工作？

        “Magic” 操作会使用多种方法检测编码数据，并推荐可用于解释数据的操作。这些方法的技术说明可在 此处 查看。

        是什么
        
                                
        一个简单直观的 Web 应用，用于分析和解码数据，无需应对复杂工具或编程语言。CyberChef 鼓励技术人员与非技术人员探索数据格式、加密与压缩。

        

                                
        为什么
        
                                
        现代世界中的数字数据形式、大小与格式千差万别——CyberChef 在一个易用的平台上帮助理解这些数据。

        


                                
        如何
        
                                
        界面以简洁为核心设计，复杂技术也可以通过拖拽轻松完成。简单函数可以组合成“配方”，实现复杂分析，并可分享给其他用户与其输入一起使用。
        
                                
        对于擅长编写代码的用户，CyberChef 提供了快速高效的原型验证方式，验证可行后即可脚本化。

        


                                
        谁
        
                                
        CyberChef 预计对网络安全与反病毒公司非常有用，也适用于学术界以及任何从事数字数据分析的个人或企业，无论是软件开发者、分析师、数学家还是解谜爱好者。

        


                                
        目标
        
                                
        希望通过 GitHub 发布 CyberChef，让贡献能够持续加入并在未来版本中发布。

        


                                
        
                                
        CyberChef 内置 数百个 实用操作，适合处理任何与互联网相关的数据任务，无论是转换时间戳格式、解压 gzipped 数据、创建 SHA3 哈希，还是解析 X.509 证书以了解颁发者。
        
                                
        它就是网络瑞士军刀。

            下载 CyberChef
            
                    
            
                    
            
                        
            
                            CyberChef 完全在浏览器中运行，无需服务器端组件，因此无论你使用官方在线版本还是下载的独立版本（前提是未被修改），你的输入数据和配方配置都不会被发送到任何地方。
                        
            
                        
            
                            有三个操作会调用外部服务：用于从 wikimedia.org 下载地图瓦片的“Show on map”操作、使用 Google 或 Cloudflare 服务解析 DNS 请求的“DNS over HTTPS”操作，以及向你配置的 URL 发起请求的“HTTP request”操作。你可以通过浏览器开发者工具（F12）的 Network 选项卡确认实际发出的网络请求。
                        
            
                        
            
                            如果你希望下载独立版 CyberChef 以在隔离网络或无/有限互联网连接的环境中使用，可在下方获取包含完整 Web 应用的 ZIP 文件。它可在本地运行或部署在 Web 服务器上，无需额外配置。
                        
            
                        
            
                            请注意，独立版本不会自动更新，这意味着在你手动重新下载新版本之前，它不会获得 bug 修复或新功能。
